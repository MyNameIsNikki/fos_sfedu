# Экзамен: Индивидуальный кейс

## Назначение
Промежуточная аттестация подтверждает освоение дисциплины в целом и проверяет способность решать сквозные прикладные задачи компьютерного зрения.

## Проверяемые компетенции и индикаторы
- DL-1, DL-3: 1.3, 3.1, 3.2, 3.3, 3.4
- Уровни: Средний, Продвинутый

## Формат
Индивидуальное задание — сквозной CV-кейс. Студент получает задачу и датасет, самостоятельно выбирает архитектуру, реализует решение, обучает модель, оценивает качество и готовит отчёт.

## Содержание кейса

Студент решает одну из сквозных задач компьютерного зрения:

1. **Классификация + детекция** — обнаружение и классификация объектов на изображениях в реальных условиях
2. **Сегментация** — выделение областей интереса на медицинских/спутниковых изображениях
3. **Генерация** — синтез изображений с заданными свойствами
4. **Мультимодальная задача** — описание изображений или ответы на вопросы по изображениям

### Этапы выполнения
1. Постановка задачи и анализ данных
2. Выбор архитектуры и обоснование
3. Реализация модели и обучение
4. Оценка качества (метрики, анализ ошибок)
5. Подготовка отчёта и защита

## Критерии оценивания (40 баллов)

| Критерий | % | Баллы |
|---|---|---|
| Полнота решения (все этапы от постановки до отчёта) | 40% | 16 |
| Качество модели (метрики, обоснованность выбора архитектуры, ablation study) | 30% | 12 |
| Анализ результатов (ошибки, ограничения, пути улучшения) | 20% | 8 |
| Оформление (структура отчёта, визуализации, читаемость кода) | 10% | 4 |

## Правила использования внешних ресурсов
- Разрешено использование любых открытых библиотек и предобученных моделей
- Запрещено копирование готовых решений целиком
- Использование LLM — только для пояснения, не для генерации решения
- Все заимствования должны быть явно указаны

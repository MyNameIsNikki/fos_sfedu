# Методические указания обучающимся

## Общие правила

### Структура дисциплины
Дисциплина «Современные методы компьютерного зрения» состоит из 4 модулей и включает 9 лабораторных работ и итоговый индивидуальный кейс.

| Модуль | Лабораторные работы | Часы |
|--------|-------------------|:----:|
| Модуль 1. Фундаментальные архитектуры и методы глубокого обучения для зрения | ЛР №1 (Виола-Джонс), ЛР №2 (CNN comparison) | 20 |
| Модуль 2. Детекция, сегментация, генерация | ЛР №3 (YOLO), ЛР №4 (U-Net), ЛР №5 (DCGAN) | 28 |
| Модуль 3. Трансформеры и метрическое обучение | ЛР №6 (ViT), ЛР №7 (Siamese) | 20 |
| Модуль 4. Продвинутые и гибридные методы | ЛР №8 (Multimodal), ЛР №9 (OCR) | 31 |

Подробное описание — в [РПД](../../docs/rpd.md).

### Система оценивания

Оценивание ведётся по 100-балльной шкале. Каждый КИМ имеет локальный максимум; итоговый балл рассчитывается по формуле нормализации (см. [Модель оценивания](../../docs/scoring-model.md)).

| Вид работы | Вклад в итог |
|---|---|
| Лабораторные работы (9 шт.) | до 60 баллов |
| Экзамен (индивидуальный кейс) | до 40 баллов |
| **Итого** | **до 100 баллов** |

### Шкала оценок

| Диапазон баллов | Оценка |
|---|---|
| 51–70 | удовлетворительно |
| 71–89 | хорошо |
| 90–100 | отлично |

### Штрафы за просрочку
- Задержка до 1 недели: −2 балла
- Задержка до 2 недель: −4 балла
- Задержка более 2 недель: −6 баллов
- Работы после окончания семестра не принимаются
- По уважительной причине (болезнь, командировка) штраф не применяется при предоставлении подтверждающего документа

## Порядок выполнения лабораторных работ

### 1. Подготовка
1. Ознакомьтесь с заданием в соответствующем КИМ
2. Изучите теоретический материал (лекции, рекомендованные ресурсы)
3. Установите необходимое ПО: Python 3.8+, Jupyter Notebook, PyTorch, OpenCV
4. Загрузите датасет (по ссылке в описании работы)

### 2. Выполнение
1. Выполните все этапы задания последовательно
2. Фиксируйте промежуточные результаты (графики, метрики)
3. Пишите код с комментариями на русском или английском языке
4. Оформляйте отчёт в Jupyter Notebook с разбивкой на разделы

### 3. Требования к отчёту
Отчёт (Jupyter Notebook) должен содержать:
- **Код** — запускаемый, с комментариями к сложным блокам
- **Визуализации** — графики, таблицы, примеры работы модели с подписями осей, легендой, заголовками
- **Комментарии к каждому графику** — краткий анализ (2–4 предложения)
- **Выводы** — сравнение методов, анализ результатов, ограничения

### 4. Защита
1. Защита проводится индивидуально, 5–7 минут на человека
2. Структура защиты:
   - Краткое представление работы (1–2 минуты): какие данные, методы, результаты
   - Ответы на вопросы преподавателя по любому из этапов работы
3. Типы вопросов:
   - **Теория:** как работает метод, математические основы
   - **Интерпретация:** почему получены такие результаты, что показывают графики
   - **Сравнение:** почему один метод лучше другого
   - **Ограничения:** какие допущения сделаны, что изменится при других условиях

### 5. Критерии оценивания ЛР

| Компонент | Баллы | Критерии |
|---|---|---|
| Письменный отчёт | 10 | Корректность кода (4), качество визуализаций (2), обоснованность выводов (2), структура (2) |
| Устная защита | 10 | Понимание теории (3), интерпретация результатов (3), сравнение методов (2), понимание ограничений (2) |

*Для ЛР №3–9 баллы могут варьироваться (см. описание конкретной работы)*

## Порядок сдачи индивидуального кейса (экзамен)

1. Вы получаете задание — сквозной CV-кейс (классификация, детекция, сегментация или генерация)
2. Выбираете архитектуру, обучаете модель, оцениваете качество
3. Готовите отчёт и защищаете решение
4. Критерии: полнота решения (40%), качество модели (30%), анализ результатов (20%), оформление (10%)

- Допускается использование LLM (ChatGPT, Copilot) для пояснения кода, отладки, генерации шаблонов
- Запрещается использовать LLM для полного решения задания

# Методические указания преподавателям по оцениванию

## Общие принципы оценивания

Каждая лабораторная работа оценивается максимум в 20 баллов (для ЛР №3, 5, 6, 8, 9 — до 30–40 баллов в зависимости от сложности). Итоговая оценка складывается из баллов за все ЛР, экзамен и дополнительные задания.

### Распределение баллов

| Вид работы | Баллы |
|---|---|
| Лабораторные работы (9 шт.) | 45–65 (суммарно) |
| Экзамен (индивидуальный кейс) | 30–40 |
| Дополнительные задания (продвинутый уровень) | 10–20 |

### Итоговая шкала

| Диапазон | Оценка |
|---|---|
| 0–50 | неудовлетворительно |
| 51–70 | удовлетворительно |
| 71–89 | хорошо |
| 90–100 | отлично |

## Структура оценивания лабораторной работы

Каждая ЛР состоит из двух компонентов:

1. **Письменный отчёт (Jupyter Notebook)** — до 10 баллов (базовый уровень)
2. **Устная защита** — до 10 баллов (базовый уровень)

### Критерии оценки письменного отчёта

| Критерий | Макс. балл | Пояснение |
|---|---|---|
| Корректность реализации | 4 | Код запускается без ошибок (`Restart & Run All`), все этапы выполнены. Синтаксическая ошибка — 0 баллов за задание. Логическая ошибка — снижение на 0.5–1.5 балла |
| Качество визуализаций | 2 | Графики подписаны (оси, легенда, заголовок). К каждому — краткий комментарий (2–4 предложения) |
| Обоснованность выводов | 2 | Студент сравнивает методы, объясняет, почему один оказался лучше |
| Структура и читаемость кода | 2 | Отчёт разбит на разделы, код с комментариями |

### Критерии оценки устной защиты

| Критерий | Макс. балл | Пояснение |
|---|---|---|
| Понимание теоретических основ | 3 | Студент объясняет суть методов, математические принципы |
| Интерпретация результатов | 3 | Студент объясняет графики, метрики, примеры работы |
| Сравнение и обоснование выбора | 2 | Студент обосновывает выбор метода/гиперпараметров |
| Понимание ограничений | 2 | Студент осознаёт допущения, ограничения методов |

### Особенности оценивания продвинутых ЛР

Для ЛР продвинутого уровня (№5, 8, 9 — уровень «Средний»):

| Работа | Макс. балл | Особенности |
|---|---|---|
| ЛР №5 (DCGAN) | 30 | +10 баллов за анализ стабильности и FID |
| ЛР №8 (Multimodal) | 40 | +20 баллов за реализацию мультимодальной архитектуры и метрики BLEU/ROUGE |
| ЛР №9 (OCR) | 40 | +20 баллов за CRNN архитектуру и CTC Loss |

## Процедура проверки

### Проверка отчёта

1. **Запуск кода.** Запустите Jupyter Notebook «с чистого листа» (Kernel → Restart & Run All). Если код падает с ошибкой — 0 баллов за задание (синтаксическая) или снижение (логическая, если студент может исправить на защите).
2. **Проверка ручной реализации.** Убедитесь, что студент реализовал алгоритмы самостоятельно, не используя готовые функции (где это запрещено условием). Использование готовых функций допускается только для верификации.
3. **Проверка визуализаций.** Оцените наличие подписей, легенд, комментариев.
4. **Проверка выводов.** Прочитайте текстовые ячейки — оцените глубину анализа и сравнение методов.
5. **Проверка структуры.** Оцените разбивку на разделы и наличие комментариев.

### Проведение защиты

1. Защита проводится индивидуально, 5–7 минут на человека
2. Студент кратко представляет работу (1–2 минуты)
3. Задайте 3–5 вопросов, охватывающих теорию, интерпретацию, сравнение, ограничения

#### Примерные вопросы по ЛР №1 (Виола-Джонса)

| Этап | Вопросы |
|---|---|
| Каскады Хаара | Что такое интегральное изображение? Как работают признаки Хаара? |
| HOG+SVM | Что такое HOG-дескриптор? Как работает SVM? |
| Сравнение | Почему Виола-Джонса лучше работает на фронтальных лицах? |
| Ограничения | Что будет, если изменить освещение? Как влияет поворот лица? |

#### Примерные вопросы по ЛР №2 (CNN comparison)

| Этап | Вопросы |
|---|---|
| Свёртки | Чем свёртка отличается от полносвязного слоя? Что такое рецептивное поле? |
| SE/ECA block | Как работает Squeeze-and-Excitation? Зачем нужен channel attention? |
| Сравнение | Почему ResNet лучше LeNet? Влияние глубины на точность |
| Ограничения | Что такое переобучение? Как с ним бороться? |

#### Примерные вопросы по ЛР №6 (ViT)

| Этап | Вопросы |
|---|---|
| Self-Attention | Как работает механизм самовнимания? Что такое Q, K, V? |
| Positional Encoding | Зачем нужны позиционные эмбеддинги? |
| ViT vs CNN | Почему ViT требует больше данных? Когда ViT лучше CNN? |
| Patch Size | Как размер патча влияет на качество и скорость? |

### Штрафы за просрочку

| Задержка | Штраф |
|---|---|
| До 1 недели | −2 балла |
| До 2 недель | −4 балла |
| Более 2 недель | −6 баллов |
| После семестра | Работа не принимается |

Штраф не применяется по уважительной причине (болезнь, командировка) при предоставлении подтверждающего документа.

## Оценка индивидуального кейса (экзамен)

| Критерий | Баллы |
|---|---|
| Полнота решения (все этапы от постановки до отчёта) | 40% |
| Качество модели (метрики, обоснованность выбора архитектуры) | 30% |
| Анализ результатов (ошибки, ограничения, пути улучшения) | 20% |
| Оформление (структура отчёта, визуализации) | 10% |

## Порядок апелляции

1. Студент может подать апелляцию в течение 3 рабочих дней после объявления оценки
2. Апелляция подаётся в письменной форме с обоснованием
3. Преподаватель повторно проверяет работу в течение 2 рабочих дней
4. При несогласии — создаётся комиссия из 2–3 преподавателей

## Рекомендации по обратной связи

- Указывайте студенту конкретные ошибки с пояснением
- Давайте рекомендации по улучшению кода, визуализаций, выводов
- Для работ с низкими баллами назначайте дополнительную консультацию
- Фиксируйте типовые ошибки для корректировки заданий в будущем

# Курсовой проект

Курсовой проект — интегральный КИМ, позволяющий проверить несколько компетенций и индикаторов в условиях комплексной практической задачи.

## Назначение
Курсовой проект завершает изучение дисциплины и проверяет способность самостоятельно выполнить законченное CV-приложение: от постановки задачи и анализа данных до реализации, обучения, оценки качества и защиты результатов.

## Проверяемые компетенции и индикаторы
- DL-1, DL-3, PL-1, FC-1
- Индикаторы: DL-1.3, DL-1.4, DL-1.9, DL-1.10, DL-1.12, DL-3.1, DL-3.2, DL-3.3, DL-3.4, PL-1.1, FC-1.2
- Уровни: Средний, Продвинутый

## Формат
Индивидуальный проект. Студент выбирает тему из предложенного списка (или предлагает свою с обоснованием), выполняет полный цикл разработки и представляет отчёт с защитой.

## Примерная тематика курсовых проектов

1. Разработка системы детекции и трекинга транспортных средств на видео с городских камер
2. Создание системы семантической сегментации медицинских изображений для диагностики
3. Разработка генеративной модели для синтеза фотореалистичных изображений интерьеров
4. Система мультимодального поиска товаров по изображению и текстовому описанию
5. Детекция аномалий на производственных линиях с помощью видеоаналитики
6. Создание системы оценки позы человека в реальном времени для фитнес-трекера
7. Разработка 3D-реконструкции объектов по нескольким 2D-изображениям
8. Система распознавания эмоций по видео лица с учётом временной последовательности
9. Автоматическая колоризация чёрно-белых исторических фотографий
10. Детекция и распознавание дорожных знаков для систем ADAS
11. Создание системы подсчёта людей в crowded-сценах
12. Разработка системы повторной идентификации человека (ReID) по видео с нескольких камер
13. Генерация синтетических данных для дообучения моделей компьютерного зрения
14. Система анализа спортивных матчей: детекция игроков и мяча, тактический анализ
15. 3D-детекция объектов для автономного вождения по данным LiDAR
16. Мультимодальная система описания изображений для помощи слабовидящим
17. Детекция deepfake-видео с использованием временных и пространственных признаков
18. Система автоматической разметки датасетов с активным обучением
19. Оптимизация нейросетевых моделей для embedded-устройств
20. Создание системы оценки качества продукции по изображениям с производственной линии

## Этапы выполнения
1. Выбор темы и постановка задачи
2. Обзор литературы и существующих подходов
3. Сбор/подготовка данных, аугментация
4. Реализация baseline и выбранной архитектуры
5. Обучение, валидация, анализ ошибок
6. Оценка качества (метрики, ablation study)
7. Подготовка отчёта и презентации
8. Защита проекта

## Критерии оценивания (40 баллов)

| Критерий | % | Баллы |
|---|---|---|
| Полнота и корректность решения (все этапы выполнены) | 30% | 12 |
| Качество модели и обоснованность выбора архитектуры | 25% | 10 |
| Анализ результатов и ablation study | 20% | 8 |
| Качество кода и воспроизводимость | 15% | 6 |

## Правила использования внешних ресурсов
- Разрешено использование любых открытых библиотек и предобученных моделей
- Запрещено копирование готовых решений целиком
- Использование LLM — только для пояснения и написания шаблонного кода, не для генерации решения
- Все заимствования должны быть явно указаны
